# Example Master Prompts

Use these as compact output-shape references. Do not copy them verbatim when the user supplies a different domain.

## Landing

```prompt
Prompt mode: concept-to-prompt
Generator target: visual website generator
Create a landing page for a premium bicycle fitting studio. The first viewport is a measurement bench with saddle, shoe, and rider-fit data cards arranged like an active fitting session. Use a restrained graphite, bone, and signal-green palette, strong product photography, and precise measurement typography. Avoid generic centered hero copy, feature-card grids, and stock cyclist silhouettes.
```

## SaaS Dashboard

```prompt
Prompt mode: concept-to-prompt
Generator target: React generator
Create a SaaS operations dashboard for field-service dispatch. The first viewport is the work surface: route map, technician status rail, dispatch queue, and exception drawer. Use dense but readable controls, shadcn/ui as accessible primitives, data-driven tables, empty/loading/error states, keyboard focus, and responsive collapse into a mobile command queue.
```

## Ecommerce

```prompt
Prompt mode: concept-to-prompt
Generator target: code agent
Create an ecommerce product page for modular desk lighting. Lead with product inspection: gallery, material swatches, beam preview, variant selector, delivery confidence, and cart state. Prioritize product media, filters, variants, trust details, image optimization, and mobile thumb reach. Avoid decorative glass panels and generic lifestyle hero sections.
```

## Portfolio / Editorial

```prompt
Prompt mode: concept-to-prompt
Generator target: visual website generator
Create an editorial portfolio for an architectural photographer. Use a contact-sheet first viewport, large type captions, project metadata, image sequencing, and a quiet monochrome palette with one warm accent. Motion should support browsing and comparison, not decorative reveals on every block.
```

## Tool UI

```prompt
Prompt mode: concept-to-prompt
Generator target: single-file HTML generator
Create a browser-based color palette inspector. The first viewport is the tool itself: upload zone, extracted swatches, contrast table, and export controls. Use semantic HTML, CSS variables, vanilla JS, accessible buttons, responsive stacked controls, and no npm dependencies.
```

## Motion Site

```prompt
Prompt mode: concept-to-prompt
Generator target: motion-heavy generator
Create a product storytelling site for a compact espresso grinder. Use GSAP ScrollTrigger for a pinned teardown sequence and Three.js for an inspectable burr assembly, with mobile fallback to static exploded images. Motion must reveal product mechanics and include reduced-motion behavior, cleanup, compressed textures, and nonblocking reading flow.
```
