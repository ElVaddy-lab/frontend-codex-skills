# Novelty Rules

Use when the user asks for a unique, premium, unusual, fresh, or absolutely different website design.

## Change the Grammar

Novelty must change the site structure, not just the theme.

Avoid default grammar:

```txt
navbar -> hero -> feature cards -> testimonials -> pricing -> CTA -> footer
```

Use alternate grammars:

- **Workbench opening**: first screen is an interactive tool surface with content embedded in controls.
- **Archive wall**: first screen is a dense visual index, catalog, or collection wall.
- **Appointment slip**: first screen feels like a ticket, receipt, form, or operating document.
- **Map room**: content is arranged as locations, routes, stations, or coordinates.
- **Control desk**: interface starts as a dashboard, console, or command surface.
- **Editorial dossier**: content unfolds like case files, pages, annotations, and evidence.
- **Product teardown**: sections are exploded views, material layers, measurements, and use cases.
- **Timeline machine**: story is navigated through time, stages, or production cycles.
- **Configurator first**: the primary interaction starts immediately, then supporting content follows.

## First Viewport Rules

- Make the brand/product/category obvious without relying only on nav text.
- Put a real experience signal in the first viewport: product state, dataset, map, editor, timeline, 3D scene, catalog, or operational UI.
- Let a hint of the next section be visible.
- Avoid centered headline plus generic CTA unless the request explicitly wants a classic landing page.

## Visual Differentiation

Change at least three of these between variants:

- Navigation model.
- First viewport composition.
- Section order.
- Content container shape.
- Motion model.
- Media type.
- Typography pairing.
- Palette logic.
- Interaction pattern.

## Anti-Patterns

- Same layout with a different industry label.
- Card grid as the answer to every section.
- Purple/blue gradient background as the main visual idea.
- Floating decorative blobs or orbs.
- Generic SaaS screenshots without product-specific UI.
- Stock-like dark hero photo with text overlay and no inspectable product detail.
