# SEO and Content Strategy

Use this reference for landing pages, portfolios, ecommerce/catalog pages, documentation, editorial sites, and public routes where search, sharing, or semantic content affects success.

## Metadata

Plan:

- page title pattern;
- meta description;
- canonical URL;
- Open Graph title, description, image;
- Twitter/social card if relevant;
- favicon and app icons;
- route-specific metadata for multi-page sites.

## Structured Data

Consider structured data when the page represents:

- organization/local business;
- product;
- article/blog post;
- FAQ;
- breadcrumb;
- event;
- portfolio project/case study.

Do not add structured data that is not represented visibly on the page.

## Content Architecture

Plan:

- semantic headings;
- primary content order;
- internal links;
- breadcrumbs when route depth needs them;
- indexable text instead of image-only text;
- content ownership: static file, CMS, markdown/content collection, API.

## Technical SEO

Consider:

- sitemap;
- robots rules;
- clean route structure;
- static rendering or server rendering for public content;
- image alt text and dimensions;
- localized routes/hreflang when relevant;
- pagination/canonical behavior for catalog pages.

## Social and Preview Assets

Plan OG images or preview assets when the page will be shared. Avoid vague "add SEO" instructions; specify which routes need which metadata.

## Output Pattern

Include SEO/content work only when the product type justifies it. For internal dashboards and tools, state that public SEO is not material unless the user says otherwise.
