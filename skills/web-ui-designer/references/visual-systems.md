# Visual Systems

Use this reference to make visual choices concrete.

## Typography

- Use a pair with clear roles: display, body, numeric/data, labels.
- For dashboards and tools, prioritize legibility and compact label systems.
- For editorial/portfolio work, allow expressive display type but keep body text calm.
- Avoid viewport-scaled fonts; use responsive layout and bounded type scales.
- Name typography roles explicitly: display, section heading, body, label, metadata, numeric/data, code/technical where relevant.

## Density Modes

- **Operational dense**: dashboards, tools, admin, SaaS workspaces. Use compact controls, clear dividers, strong alignment, and predictable rhythm.
- **Balanced product**: marketing plus app previews. Use enough whitespace for hierarchy but keep proof and actions visible.
- **Editorial spacious**: portfolio, content, launch, brand-led pages. Use larger media rhythm and more generous reading space.
- **Commerce inspectable**: product/catalog flows. Reserve room for media, specs, variants, filters, price, trust, and cart feedback.

## Spacing and Layout Rhythm

- Use a spacing scale rather than one-off gaps.
- Decide density from task: dense for operations, spacious for editorial or brand-led pages.
- Use full-width bands or unframed layouts for page sections.
- Use cards only for repeated items, modals, and genuinely framed tools.

## Surfaces

Choose surfaces by function:

- Panels for tools and data.
- Bands for narrative sections.
- Tables/lists for comparison and scanning.
- Drawers/modals for temporary focused tasks.
- Canvas/workbench for creation or configuration.

Avoid generic glass panels unless transparency communicates layered product state or media depth.

## Media and Icons

- Use media that reveals the actual product, work, place, object, or state.
- Prefer inspectable product images, interface views, diagrams, maps, or generated visual systems over stock-like atmospheric imagery.
- Use icons for recognizable actions and controls.
- Keep icons consistent in stroke, size, and semantic role.

## Media Direction

- Product/site pages should show the real product, UI state, object, place, work sample, or workflow output.
- Dashboards should show meaningful data states, not decorative charts.
- Tool UI should show the work surface or result preview.
- Ecommerce should make product inspection possible through stable media, variants, zoom/detail, and comparison.
- Portfolio/editorial should make media sequencing part of the story, not a generic image grid.

## Motion

Motion should clarify:

- Hierarchy.
- Spatial relationship.
- State change.
- Progress.
- Product behavior.

Avoid motion that delays reading, hides important content, or makes operational UI feel unstable.
