# Visual Accessibility, Contrast, and Motion

## Contrast

Check:

- normal text;
- large text;
- links in body text;
- placeholder and helper text;
- disabled text where it still communicates information;
- focus indicators;
- icons and graphical controls;
- form borders and selected states;
- charts and status indicators.

Report foreground, background, text size/weight when relevant, ratio when measured, threshold, and state.

## Use of Color

Color must not be the only way to communicate state, errors, required fields, selection, chart categories, or availability. Check labels, icons, patterns, text, or shape alternatives.

## Target Size and Pointer Input

Check touch targets, spacing between adjacent targets, icon buttons, inline links, carousel controls, table row actions, and dense toolbars.

## Zoom, Reflow, and Text Spacing

Check:

- 200% zoom or equivalent reflow where practical;
- no two-dimensional scrolling for normal content unless the content type requires it;
- text does not clip or overlap;
- controls remain usable;
- text spacing changes do not break layout.

## Motion and Timing

Check:

- reduced-motion support;
- animation from interactions;
- auto-playing motion;
- carousels and moving content can pause/stop/hide;
- flashing risk;
- time limits can be extended or disabled when relevant.

## Hover and Focus Content

Tooltips, popovers, hover cards, and focus-revealed content should be dismissible, hoverable, and persistent when required. They must not obscure critical content without a way out.
