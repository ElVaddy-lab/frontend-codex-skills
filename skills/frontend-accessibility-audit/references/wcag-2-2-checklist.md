# WCAG 2.2 Checklist

Use WCAG 2.2 as the default reference. Prefer WCAG 2.2 AA unless the user asks for another level.

## Principles

WCAG 2.2 is organized around four principles:

- **Perceivable**: users can perceive content and alternatives.
- **Operable**: users can operate controls and navigate.
- **Understandable**: users can understand content and interactions.
- **Robust**: content works with user agents and assistive technologies.

## Practical AA Focus Areas

Prioritize these common Web/UI areas:

- non-text content and meaningful alternatives;
- captions/audio descriptions when media exists;
- information and relationships;
- meaningful sequence;
- use of color;
- text contrast;
- resize text, reflow, and text spacing;
- non-text contrast;
- content on hover or focus;
- keyboard access and no keyboard trap;
- pause, stop, hide for moving content;
- flashing/seizure risk;
- bypass blocks;
- page titles;
- focus order;
- link purpose;
- headings and labels;
- focus visible;
- focus not obscured;
- pointer gestures and pointer cancellation;
- label in name;
- dragging alternatives;
- target size minimum;
- language of page and parts;
- predictable behavior;
- consistent navigation and identification;
- consistent help;
- error identification;
- labels or instructions;
- error suggestions;
- error prevention for sensitive data;
- redundant entry;
- accessible authentication;
- name, role, value;
- status messages.

## WCAG 2.2 Additions to Watch

For modern interfaces, pay special attention to:

- focus not obscured;
- focus appearance;
- dragging movements;
- target size minimum;
- consistent help;
- redundant entry;
- accessible authentication.

## Mapping Rules

- Map a finding to WCAG only when the relationship is concrete.
- Do not claim full conformance unless all relevant criteria and complete processes were tested.
- Use `WCAG relevance` instead of `WCAG violation` when evidence is incomplete.
- Mark third-party or embedded content limitations clearly.
- If a user asks for legal compliance, state that the audit is technical evidence, not legal advice.

## Conformance Caution

WCAG conformance depends on full pages, complete processes, accessibility-supported technologies, and non-interference. A component-level or route-level audit can support remediation, but it does not prove product-wide conformance by itself.
